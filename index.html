Typography
foundation.

The eCare typography system is built on a single typeface — YouNik — and a tight set of role-based tokens that snap between an App / Mobile and a Desktop scale at 1024 px. Every component reads from this surface, so the type stays consistent from a 12-pixel label on a card to a 56-pixel block on a marketing page.

This page is the canonical reference. Each token is shown in isolation with its size, weight, leading and tracking. Values are generated from the Figma variable export — when the Figma file changes, regenerate the tokens here.

06 · Responsive behaviour

Snap at 1024 px.

Type doesn't interpolate — it switches modes at a single breakpoint. Below 1024 px every scaling role uses its App / Mobile value; at or above 1024 px it snaps to its Desktop value. Drag the viewport below to see the change.

08 · Usage

How to use it.

Two ways to reach the type system from product code — either pull the utility class onto a semantic element, or reference the underlying CSS variables directly when you need something custom.

Utility class

<!-- Drop the class on a semantic tag -->
<h1 class="type-h1">
  Manage your care plan.
</h1>

<p class="type-body-large">
  Default reading text.
</p>

CSS variables

/* Compose a custom block */
.callout {
  font-family: var(--type-h2-family);
  font-size:   var(--type-h2-size);
  font-weight: var(--type-h2-weight);
  line-height: var(--type-h2-leading);
  letter-spacing: var(--type-h2-tracking);
}

File layout

/* Load order matters — tokens first, then utility classes */
<link rel="stylesheet" href="css/tokens.css">     // font-faces + CSS variables
<link rel="stylesheet" href="css/typography.css"> // .type-* utility classes
