Card
component.

The eCare card is the primary surface for grouping related content. It appears throughout the app as the container for appointments, prescriptions, care plan summaries and informational panels — wherever a visual unit needs to stand apart from the background.

The component is a pure container: one variant, three Figma tokens. Background, radius and padding are fixed by design — what goes inside the slot is entirely up to the consuming component. Padding snaps from 20 px on mobile to 24 px on desktop.

03 · Anatomy

What's inside.

Two parts. The container handles all visual properties — the slot receives any content without constraints.

A
Container

The card surface. Sets background to the product-cards warm off-white, border-radius to 16 px, and padding to the responsive padding token. Also defines a flex-column layout with matching gap for direct children.

B
Content slot

A full-width, unconstrained area. Place headings, body text, links, status labels or nested components here. Internal spacing is defined by the slotted content — or use the card's own flex gap by placing content as direct children.

04 · Tokens

Three Figma variables.

All three tokens come directly from the Figma variable export — no overrides or aliases. Background and radius are constant across themes and breakpoints; padding snaps at 1024 px.

Token Figma variable Mobile / Light Desktop / Dark
--color-bg-product-cards Background Colors/Background Product Cards --color-bg-product-cards --color-bg-product-cards
--border-radius-16 Border/Border Radius/16 16 px — (constant)
--cards-padding-product Cards/Padding/Product Cards 20 px 24 px
05 · Code

How to build it.

A single <div class="card"> wrapper. No required children — place any content directly inside. The flex-column layout and gap are already set by the component.

HTML

<!-- Card with slotted content -->
<div class="card">

  <!-- Group tightly related items in a sub-div -->
  <div>
    <p class="type-h6">Section label</p>
    <p class="type-h4">Card heading</p>
  </div>

  <!-- Direct children receive the card's flex gap -->
  <p class="type-body-small">Supporting text.</p>
  <a class="hlink hlink--no-underline" href="…">
    <span class="hlink__text">Action label</span>
  </a>

</div>

CSS

.card {
  background:     var(--color-bg-product-cards);   /* rgba(240,235,220,0.6) */
  border-radius:  var(--border-radius-16);          /* 16 px */
  padding:        var(--cards-padding-product);      /* 20 px → 24 px */
  display: flex;
  flex-direction: column;
  gap:            var(--cards-padding-product);
}

/* css/tokens.css — padding snaps at 1024 px */
:root                           { --cards-padding-product: 1.25rem; } /* 20 px */
@media (min-width: 1024px) {
  :root                         { --cards-padding-product: 1.5rem;  } /* 24 px */
}
