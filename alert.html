Alert
component.

Alerts surface contextual feedback at the right moment — a prescription sent, a warning about a missed dose, a lab result that needs attention. They appear in-page, anchored to the content that triggered them, never as an overlay.

Four semantic variants map to signal strength: success for positive outcomes, information for neutral context, warning for caution states, and error for urgent or destructive situations. Each has a dedicated background colour and icon.

03 · Anatomy

What's inside.

Every alert is built from four slots. Only the body is required — headline, icon and link compose freely on top.

Headline — short and specific

Body text. Keep to one or two sentences — if more is needed, link to a detail view rather than expanding the alert.

Link Link 2
A
Icon

Semantic glyph for the variant. Always 24 × 24 px, aria-hidden. Do not swap for a custom icon — the shape carries meaning.

B
Headline

Optional. 4–8 words. Omit when the body text stands alone. Uses Font/H5 — 16 px / 18 px, weight 500.

C
Body

Required. One or two sentences. Uses Font/Body Small — 14 px / 16 px, weight 400.

D
Link

Optional. Underlined text CTA. Uses Font/Button Medium — 14 px / 15 px, tracking 0.2 px.

E
Link 2

Optional second link. Maximum two links per alert. On desktop both sit side by side to the right; on mobile they stack below the body text.

05 · Usage

When to use what.

The variant is a semantic choice — not an aesthetic one. Map the system state to the variant that accurately represents it. When in doubt, default to information.

Variant Colour Use when Do not use for ARIA
Success --color-alert-success Confirming a completed action: saved, sent, booked Ongoing states — success is a moment, not a status role="note"
Information --color-alert-info Neutral context, tips, non-urgent status updates Anything that requires action or has an outcome role="note"
Warning --color-alert-warning Situations that need attention but aren't yet broken Errors that have already occurred role="note"
Error --color-alert-error Errors, data loss risk, urgent health alerts Routine updates — reserve for genuine urgency role="alert"
06 · Code

How to build it.

A single element with a BEM modifier for the variant. Only .alert__body with at least .alert__text is required. Swap role="note" for role="alert" on error alerts so screen readers announce them immediately.

HTML structure

<!-- role="alert" announces immediately · role="note" is passive -->
<div class="alert alert--success" role="note">

  <div class="alert__icon" aria-hidden="true">…svg…</div>    <!-- optional -->

  <div class="alert__content">

    <div class="alert__body">
      <p class="alert__title">Headline</p>      <!-- optional -->
      <p class="alert__text">Body text.</p>       <!-- required -->
    </div>

    <a class="hlink alert__link" href="…">    <!-- optional -->
      <span class="hlink__text">Link</span>
    </a>
    <a class="hlink alert__link" href="…">  <!-- optional second link -->
      <span class="hlink__text">Link 2</span>
    </a>

  </div>

</div>

Variant modifiers

.alert--success      /* var(--color-alert-success) · role="note"  */
.alert--info         /* var(--color-alert-info)    · role="note"  */
.alert--warning      /* var(--color-alert-warning) · role="note"  */
.alert--error        /* var(--color-alert-error)   · role="alert" */
