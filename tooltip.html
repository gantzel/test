Tooltip
component.

The Tooltip is a compact floating label that reveals supplementary information when a user hovers or focuses an element. It uses a dark rounded pill with 12 px H7 text and an optional directional arrow that anchors it visually to the trigger.

Five variants cover all placement needs: no arrow, or an arrow pointing up, down, left, or right. Keep tooltip text concise — one short phrase. Never put interactive content or essential information inside a tooltip.

02 · Variants

Five variants.

The arrow direction indicates which side of the trigger element the tooltip is anchored to. "None" is used when placement is handled purely by position — no visual pointer is needed.

03 · Anatomy

Two parts.

My Tooltip
A
Body

A dark rounded pill with padding: 4px 8px and border-radius: --border-radius-4 (4 px). Background is --color-text-dark-primary; text uses --color-text-light-primary at H7 — 12 px / 14 px, weight 500. Flips to light in dark mode.

B
Arrow

A CSS border-triangle, 10 × 6 px for up/down variants and 6 × 10 px for left/right. Colour matches the body background. Omitted in the None variant.

04 · Tokens

Figma variables.

The tooltip always contrasts with the page background — dark in light mode, light in dark mode. No separate dark mode colour tokens are needed; the component overrides them directly.

Token Figma variable Light Dark
--color-text-dark-primary Text / Dark Colors / Text Primary Body background Body text
--color-text-light-primary Text / Light Colors / Text Primary Body text Body background
--border-radius-4 Border / Border Radius / 4 4 px — constant
--type-h7-size Font / Size / H7 12 px — constant
--type-h7-weight Font / Weight / Medium 500 — constant
05 · Code

How to build it.

A role="tooltip" wrapper with a .tooltip__body and an optional .tooltip__arrow. The modifier class sets both the flex direction and the arrow orientation. Positioning relative to the trigger is handled by the consuming component.

HTML structure

<!-- No arrow -->
<div class="tooltip" role="tooltip">
  <div class="tooltip__body">Label text</div>
</div>

<!-- Arrow up — tooltip below trigger -->
<div class="tooltip tooltip--arrow-up" role="tooltip">
  <div class="tooltip__arrow"></div>
  <div class="tooltip__body">Label text</div>
</div>

<!-- Arrow down — tooltip above trigger -->
<div class="tooltip tooltip--arrow-down" role="tooltip">
  <div class="tooltip__body">Label text</div>
  <div class="tooltip__arrow"></div>
</div>

<!-- Arrow left — tooltip right of trigger -->
<div class="tooltip tooltip--arrow-left" role="tooltip">
  <div class="tooltip__arrow"></div>
  <div class="tooltip__body">Label text</div>
</div>

<!-- Arrow right — tooltip left of trigger -->
<div class="tooltip tooltip--arrow-right" role="tooltip">
  <div class="tooltip__body">Label text</div>
  <div class="tooltip__arrow"></div>
</div>

Modifiers

/* On .tooltip */
.tooltip--arrow-up     /* arrow at top    — tooltip below trigger  */
.tooltip--arrow-down   /* arrow at bottom — tooltip above trigger  */
.tooltip--arrow-left   /* arrow at left   — tooltip right of trigger */
.tooltip--arrow-right  /* arrow at right  — tooltip left of trigger  */

Accessibility

<!-- Link trigger to tooltip via aria-describedby -->
<button aria-describedby="tip-1">Save</button>
<div class="tooltip tooltip--arrow-down"
     id="tip-1" role="tooltip">
  <div class="tooltip__body">Saves your progress</div>
  <div class="tooltip__arrow"></div>
</div>
