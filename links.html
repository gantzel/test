Link
component.

Links navigate users to related content or trigger an action within a flow. They sit inline in body copy or standalone as a labelled call to action — always using clear, descriptive text rather than generic labels.

Two props control appearance: underline determines whether a 1 px rule sits 3 px below the text, and disabled reduces opacity to signal that the action is temporarily unavailable. No variants or colour modifiers.

03 · Anatomy

What's inside.

Two parts. Only the label is required — the underline is toggled on or off as a prop.

Link text
A
Label

Required. Descriptive text for the destination or action. Uses Font/Components/Button Large — 16 px / 18 px, weight 500, tracking 0.2 px.

B
Underline

Optional. A 1 px horizontal rule rendered 3 px below the label baseline. Matches the label colour — including the disabled state. Hidden when underline=false.

04 · Usage

When to use what.

The underline improves scannability when a link appears inside a paragraph. Omit it only when the interactive nature is already clear from context — e.g. a navigation row or a clearly labelled CTA button row.

State When to use Notes
Link Inline in body copy; any link where destination is not obvious from layout Default — use this unless you have a reason to remove the underline
Link Standalone CTAs in cards, alerts, or navigation rows where the interactive role is self-evident Do not remove the underline from links embedded in paragraphs
Link Action is temporarily unavailable — e.g. before a required step is complete Pair with a tooltip explaining why. Remove from tab order with tabindex="-1"
Do
  • Write descriptive labels — "View prescription" not "Click here"
  • Keep labels concise — 2–5 words
  • Use aria-disabled="true" and tabindex="-1" on disabled links
  • Underline links inside body text
  • Use a single link per context when possible
Don't
  • Use "here", "this" or "read more" as the only label text
  • Remove underlines from inline paragraph links
  • Use links for destructive or irreversible actions — use a button
  • Style links to look like buttons or vice versa
  • Disable a link without explaining why
05 · Code

How to build it.

An <a> element wrapping a single label span. The underline is drawn via text-underline-offset — positioned exactly 3 px from the baseline to match Figma. BEM modifiers control state.

HTML

<!-- Default: underline on, enabled -->
<a class="hlink" href="…">
  <span class="hlink__text">Link text</span>
</a>

<!-- No underline -->
<a class="hlink hlink--no-underline" href="…">
  <span class="hlink__text">Link text</span>
</a>

<!-- Disabled -->
<a class="hlink hlink--disabled" href="#"
   aria-disabled="true" tabindex="-1">
  <span class="hlink__text">Link text</span>
</a>

CSS

.hlink {
  display: inline-block;
  text-decoration: none;
  color: var(--color-text-dark-primary);        /* rgba(0,0,0,0.8) */
  white-space: nowrap;
  cursor: pointer;
}

.hlink__text {
  font-family:    var(--font-family-sans);               /* Font/Style/YouNik */
  font-size:      var(--type-button-large-size);         /* 16px */
  font-weight:    var(--type-button-large-weight);       /* 500 */
  line-height:    var(--type-button-large-leading);      /* 18px */
  letter-spacing: 0.2px;
  text-decoration:           underline;
  text-decoration-thickness: 1px;
  text-underline-offset:     3px;                          /* Link/Spacing — from baseline */
  text-decoration-color:     currentColor;
}

.hlink--no-underline .hlink__text { text-decoration: none; }

.hlink--disabled {
  color: var(--color-disabled);               /* rgba(0,0,0,0.38) */
  cursor: default;
  pointer-events: none;
}

.hlink:hover:not(.hlink--disabled) { opacity: 0.65; }
