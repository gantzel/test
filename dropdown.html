Dropdown
component.

The Dropdown is a single-line select control that lets users pick one option from a list. Like the Input Field it uses a floating label that sits inside the field and rises to sit on the border when the field is open or has a value.

An optional search icon can be placed on the left to signal that the field supports filtering. The right-side chevron is always present and rotates 180° when the menu is open.

04 · Anatomy

Four parts.

Label Selected value

Helper text

A
Wrapper

The bordered container. min-height: 48px, border-radius: --border-radius-8 (8 px). Default border is 1 px --color-border-default; active state uses 2 px --color-border-active (brand green in dark mode). When open, only the top, left, and right sides show the 2 px border.

B
Floating label

Positioned absolute, starts centered vertically at H6 size (14 px). On focus or when the field has a value, transitions to top: 0 at 12 px / 0.3 px tracking, sitting on the border with background: --color-bg-frame.

C
Value text

The selected option displayed as H6 Medium — 14 px / weight 500. Uses --color-text-dark-secondary as placeholder colour and --color-text-dark-primary once a value is set. Add an optional .dropdown__icon before this slot for a left-side search icon.

D
Chevron

Always present. A 20 × 20 px arrow icon that rotates 180° when the menu is open via transform: rotate(180deg) on .dropdown--open.

E
Helper text · optional

An optional .dropdown__helper line below the wrapper provides extra context. Required in the Error state, where it renders in --color-sys-error to describe the validation message.

05 · Tokens

Figma variables.

The active border switches to brand green in dark mode — the same token as Input Field. All other tokens adapt automatically.

06 · Code

How to build it.

The dropdown is a styled div — not a native <select>. Apply state modifier classes via JavaScript as the user interacts. Keyboard and ARIA handling is left to the implementing component.

HTML — standard

<!-- Default / empty -->
<div class="dropdown">
  <div class="dropdown__wrapper">
    <span class="dropdown__label">Label</span>
    <span class="dropdown__value"></span>
    <div class="dropdown__chevron"><!-- chevron SVG --></div>
  </div>
  <p class="dropdown__helper">Helper text</p>
</div>

<!-- Has value -->
<div class="dropdown dropdown--has-value">
  <div class="dropdown__wrapper">
    <span class="dropdown__label">Label</span>
    <span class="dropdown__value">Selected option</span>
    <div class="dropdown__chevron"><!-- chevron SVG --></div>
  </div>
  <p class="dropdown__helper">Helper text</p>
</div>

HTML — with optional icon

<div class="dropdown dropdown--has-icon">
  <div class="dropdown__wrapper">
    <button class="dropdown__icon" type="button"
            aria-label="Search">
      <!-- 20×20 search SVG -->
    </button>
    <span class="dropdown__label">Label</span>
    <span class="dropdown__value"></span>
    <div class="dropdown__chevron"><!-- chevron SVG --></div>
  </div>
  <p class="dropdown__helper">Helper text</p>
</div>

HTML — open/expanded

<div class="dropdown dropdown--open dropdown--has-value">
  <div class="dropdown__wrapper">
    <span class="dropdown__label">Label</span>
    <span class="dropdown__value">Selected option</span>
    <div class="dropdown__chevron"><!-- chevron SVG --></div>
  </div>
  <div class="dropdown__menu">
    <button class="dropdown__option" type="button">Option 1</button>
    <button class="dropdown__option" type="button">Option 2</button>
    <!-- … -->
  </div>
  <p class="dropdown__helper">Helper text</p>
</div>

State modifiers

/* Applied to .dropdown */
.dropdown--active     /* Focused — 2px border, label floated  */
.dropdown--open       /* Menu visible — chevron rotated 180°  */
.dropdown--has-value  /* Value selected — label floated       */
.dropdown--error      /* Red border + red helper + red label  */
.dropdown--disabled   /* All text disabled, pointer-events: none */
.dropdown--has-icon   /* Left icon present — offsets unfloated label */
