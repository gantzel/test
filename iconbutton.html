Icon Button
component.

The Icon Button is a compact call-to-action that holds a single icon with no text label. It shares the same five styles and pill shape as the Button component, and is used wherever a text label would be redundant or space is constrained.

Three sizes — Extra Large (36 px), Large (32 px), and Small (28 px) — cover the full range from primary actions in a toolbar to secondary controls in compact list rows. Always pair with an aria-label so the action is announced to screen readers.

02 · Styles

Five styles.

Each style signals a different level of emphasis. Use Primary for the dominant action, Secondary for supporting actions, and the outline and tertiary variants for lower-emphasis or contextual controls. Each card shows Extra Large, Large, and Small.

Primary

Green fill. Reserved for the single primary CTA. Icon is always-black.

.icon-btn--primary
Secondary

Dark fill with light icon. Use alongside a Primary action.

.icon-btn--secondary
Outline Primary

2 px border, transparent background. Tertiary emphasis.

.icon-btn--outline-primary
Outline Secondary

1 px divider-colour border. Subtle, inline use in lists and cards.

.icon-btn--outline-secondary
Tertiary

No background, no border. Ghost style for low-emphasis icons.

.icon-btn--tertiary
03 · States

Four states.

All five styles support the same four states. Hover and Pressed apply overlay colours from the token system. Disabled removes interaction and reduces contrast. Shown at Extra Large size.

04 · Anatomy

Two parts.

A
Container

The button element itself. A square with equal width and height, determined by padding + icon size: Extra Large 36 px (6 px padding · 24 px icon), Large 32 px (6 px padding · 20 px icon), Small 28 px (5 px padding · 18 px icon). border-radius: --border-radius-round (512 px) gives the circle shape.

B
Icon

A single SVG icon centred inside the container. Size adapts with the size modifier: 24 px (XL), 20 px (Large), 18 px (Small). The icon inherits currentColor — set by the style modifier — so it automatically adapts to all themes and states. Always add aria-hidden="true" on the SVG and aria-label on the button.

06 · Code

How to build it.

Compose two BEM modifiers on a single <button> — one for style and one for size. Always add aria-label describing the action, and aria-hidden="true" on the inner SVG.

HTML structure

<button class="icon-btn icon-btn--primary icon-btn--xl" aria-label="Search">
  <svg class="icon-btn__icon" viewBox="0 0 24 24" aria-hidden="true">
    <!-- icon path(s) using stroke/fill="currentColor" -->
  </svg>
</button>

<!-- Disabled -->
<button class="icon-btn icon-btn--primary icon-btn--xl icon-btn--inactive" disabled aria-label="Search">
  <svg class="icon-btn__icon" viewBox="0 0 24 24" aria-hidden="true">…</svg>
</button>

Modifiers

/* Style — pick one */
.icon-btn--primary          /* green fill · always-black icon */
.icon-btn--secondary        /* dark fill  · light icon */
.icon-btn--outline-primary  /* 2 px border · no fill */
.icon-btn--outline-secondary /* 1 px divider border · no fill */
.icon-btn--tertiary         /* no background, no border */

/* Size — pick one */
.icon-btn--xl   /* 36 × 36 px — 24 px icon + 6 px padding */
.icon-btn--lg   /* 32 × 32 px — 20 px icon + 6 px padding */
.icon-btn--sm   /* 28 × 28 px — 18 px icon + 5 px padding */

/* State */
.icon-btn--inactive  /* disabled · add disabled attribute too */
